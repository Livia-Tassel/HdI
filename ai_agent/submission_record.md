# 智能体协作记录（提交版）

## 1. 协作目标
本项目围绕赛题《健康数据洞察》开展，AI 智能体的角色不是替代团队完成结论判断，而是协助完成以下四类工作：

1. 读取赛题要求并拆解为可执行任务；
2. 结合真实 provided 数据结构筛选可落地方法；
3. 参与代码生成、调试与静态结果导出；
4. 辅助组织结果解释、英文仪表盘与提交材料。

## 2. 协作内容与实际贡献

### 2.1 需求理解
- 协助重新阅读 `resources/instruction.pdf`，确认比赛最低提交材料包括：
  - 主题分析报告
  - 数据处理源程序
  - 展示视频
  - 自增数据集（若有）
  - 智能体交互记录
- 将赛题任务整理为三个主维度：
  - Dimension 1：疾病谱演变与健康差距
  - Dimension 2：风险因素归因与情景模拟
  - Dimension 3：资源缺口、效率象限与资源优化

### 2.2 方法建议与取舍
- AI 在早期曾给出 GHRI、DEA、严格 PAF 等方法建议，用于扩展思路与方案比较。
- 团队随后基于真实数据可得性做了收敛：
  - **最终采用**
    - 三大类疾病谱趋势与健康指标解释
    - 风险归因死亡贡献分析（attributable share）
    - 资源缺口、投入-产出四象限与约束优化
  - **未作为最终主线采用**
    - GHRI 综合指数
    - 依赖额外强假设的 DEA/GHRI 脚手架
    - 需要暴露率与 RR 参数的严格 PAF 估计
- 该取舍过程体现了 AI 的“方法备选”作用，而最终是否采用由团队依据数据条件和实现结果决定。

### 2.3 代码生成与调试
- AI 参与的关键代码工作包括：
  - 真实数据清洗与国家映射修复
  - 主分析面板、资源面板与中国省级面板构建
  - 三个维度的静态 API 结果导出
  - 英文静态仪表盘重构与离线化
  - Dimension 3 Optimization Lab 场景引擎导出
- 重点调试内容包括：
  - 中文列名清洗导致字段丢失
  - 中文国家名到 ISO3 的误判
  - `/api/v1/dim3/optimization` 查询参数未真正生效
  - `/api/v1/dim2/shapley` 对未实现过滤条件的接口诚实性
  - 前端对远程脚本、远程旗帜资源和启动遮罩的依赖

### 2.4 结果解释与交付包装
- AI 协助整理：
  - `reports/report.tex` 中的中文分析叙述
  - `video/demo_script.md` 的视频讲解路径
  - 英文仪表盘中的摘要卡片、国家详情与 Optimization Lab 文案
  - `data/catalog.yaml` 与本协作记录等提交辅助材料

## 3. 扩展功能说明
在赛题原始要求之外，本项目新增了一个与主题强相关的扩展功能：

**Optimization Lab**
- 面向 Dimension 3 的资源再分配实验室；
- 支持两类目标：
  - `max_output`：最大化总体健康产出
  - `maximin`：优先抬升最低健康产出
- 支持三档预算情景：
  - `0.9x`
  - `1.0x`
  - `1.1x`
- 以静态场景结果导出到 API 与英文仪表盘中，便于离线演示与横向比较。

该扩展直接服务于赛题第 3 维中“设定不同目标、模拟卫生资源最优再分配方案”的要求，同时把原本单一结果拓展为可切换、可解释、可展示的决策实验界面。

## 4. 证据文件对应关系
- `ai_agent/workflow.md`
  - 记录本次真实协作流程与阶段性贡献。
- `ai_agent/interaction_logs/*.json`
  - 保存部分 AI 交互样例，主要体现早期方案探索与方法比较。
- `src/hdi/`
  - 保存最终被采用的实现代码，应作为最终结果的唯一技术依据。
- `api_output/`、`dashboard/data/`、`reports/`
  - 保存最终导出的静态结果、仪表盘数据与报告图表。

## 5. 诚实性声明
- 本项目保留了 AI 提出的探索性方案记录，但**只有已经在代码、结果文件与报告中落地的内容才构成正式提交成果**。
- 因此，若某些 JSON 交互样例中提到 GHRI、DEA 或其他未落地方法，应理解为“方案讨论痕迹”，而非最终结果本身。
- 最终分析结论、图表和静态接口输出均以当前仓库中的实现与导出文件为准。
