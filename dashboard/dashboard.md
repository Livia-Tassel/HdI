你是一位资深数据可视化产品设计师、前端体验审美顾问和竞赛展示型仪表盘设计总监。请继续重构并美化当前 `dashboard/` 静态网站，但本轮只完成一个明确阶段，不要一次性大改全部页面。你的目标不是做炫技型界面，而是把它打磨成一个面向评委浏览、解释、交互和决策都足够顺滑的全球健康洞察仪表盘。

当前网站主题是“健康洞察 / Health Insight”，已经从深色玻璃风改造到浅色数据仪表盘风格，但仍然存在设计完成度不足、局部装饰感偏重、视觉层级不够克制、国家选中高亮颜色与地图色带冲突、控件细节不够精致、评委交互路径还可以更顺畅的问题。请以“高级公共卫生数据产品 + 竞赛答辩展示 + 数据新闻式叙事”的标准继续设计。

本轮重构必须围绕“浅色、清晰、专业、可信、顺滑、有主题记忆点”推进。整体感觉参考 WHO、Our World in Data、World Bank DataBank、The Economist 数据可视化页面、Apple Human Interface Guidelines 中的轻量信息层级，而不是普通后台管理系统模板。界面应该让评委第一眼知道这是一个严肃的全球健康与卫生资源公平分析系统，同时在细节上感到精致、平静、现代。

核心设计原则：

1. 数据优先，而不是装饰优先。所有视觉效果都必须服务于阅读、比较、定位和解释。
2. 浅色基底必须保持舒适，不要纯白刺眼，也不要重新回到深色或强玻璃拟态。
3. 页面需要有高级感，但不能牺牲实用性。不要添加会干扰地图、图表、排名和指标阅读的装饰。
4. 中文文字、数字、图例、标签和按钮必须清晰。不要使用过小、过淡、过挤的文字。
5. 每个组件都应该像一个成熟数据产品的一部分：对齐准确、间距一致、状态明确、反馈柔和。
6. 交互要让评委“少想一步”：维度切换、国家搜索、地图点击、排名点击、弹窗查看都应该有明确反馈。
7. 国家/地区选中高亮绝对不要再使用蓝色或青蓝色，因为会和地图主色带冲突。请改成高对比、非蓝色的全局选择色。

全局视觉方向：

使用“温和浅灰背景 + 白色功能卡片 + 克制边线 + 少量医学蓝绿品牌色 + 一个非蓝色选择高亮色”的设计系统。页面背景不应是单调纯色，可以保留极淡的数据网格或浅色地图纹理感，但透明度必须非常低，只用于营造数据产品氛围，不能抢地图视觉。卡片不应像漂浮玩具，也不应大量使用强阴影。高级感来自精确布局、留白、层级、细线、色彩克制和动效节奏。

1. 地图上被选中的国家或省份使用 `#D81B60` 作为主描边，线宽明显高于普通边界。

5. 搜索下拉列表中的当前选中项可以继续使用品牌蓝作为控件焦点，但最终“国家已选中”的状态要有洋红标记，例如左侧细条或小圆点。

顶部 masthead 设计要求：

控件栏设计要求：

1. 控件栏是评委探索的入口，必须比现在更有秩序。
2. 维度切换按钮建议设计成“分段标签 + 下划线/底部强调线”的形式，减少圆角胶囊过多造成的拥挤感。
3. 当前维度 active 状态要明确，但不要全蓝大面积填充。普通维度可用品牌蓝细线；中国大陆聚焦可用风险红或中国主题暖红细线；健康全景可用健康绿细线。
4. 控件 label 保持小号大写/字距风格，但中文不要过度拉开，避免阅读疲劳。
5. 下拉框、搜索框、预算选择等高度一致，边线一致，focus 状态统一。
6. 搜索 combobox 是关键交互：输入框左侧搜索图标要细致，清除按钮要小而可点，列表打开时要有清晰边界和阴影。

主地图卡片设计要求：

1. 地图是页面主角。地图卡片要比其他卡片更稳重，留白充足，标题区和地图区边界清楚。

8. 地图缩放/拖拽如果不是核心功能，可以保持关闭，避免误操作。若有重置按钮，要在 hover 或选择后自然出现，不要常驻干扰。

中国大陆省级地图设计要求：

国家概况侧栏设计要求：

1. 国家概况是“解释当前选中对象”的区域，不应只是指标堆叠。

5. 指标卡片的左边色条应与指标含义相关：寿命/覆盖用蓝绿，风险用红橙，资源用绿/紫，当前选中状态用洋红。

图表卡片设计要求：

1. 图表必须看起来像一个统一系统，不要每个图表样式不一致。
2. 所有 Plotly 图表使用浅色 paper/plot 背景，网格线使用极浅灰，轴线轻，tick 字体清晰。
3. 图表标题由卡片 header 承担，Plotly 内部标题尽量减少重复。
4. hover label 白底深字，边线浅灰，必要时使用洋红强调当前国家。
5. 当前国家/省份的数据点、线、标记不要用蓝色，统一使用洋红。
6. 其他国家或背景点使用中性灰、区域色或半透明，避免与当前对象抢视觉。
7. 条形图中当前对象条形可用洋红描边或末端圆点，不要全条大面积洋红，除非它本身是唯一主角。
8. 趋势图中当前国家线可使用洋红，全球/区域参考线使用灰蓝或浅灰。
9. 象限图中当前点使用洋红填充 + 白色描边 + 轻微阴影，其他点根据象限使用克制色。
10. 图例位置要稳定，移动端不要覆盖数据。

核心洞察卡片设计要求：

1. 核心洞察应该像“评委可直接引用的结论区域”，不是普通列表。
2. 第一段 lede 使用稍大的正文，解释当前维度最重要发现。

5. 政策建议块可以使用浅暖色背景，但边线和文字必须克制。不要使用大面积黄色造成廉价感。
6. 推荐策略中的“优先行动”应比正文略强调，但不要像警告框。

Spotlight 国家弹窗设计要求：

1. 弹窗是评委点击国家后的深度阅读空间，应该像一张精致的国家 profile sheet。
2. 桌面端从右侧滑入可以保留，但速度要自然，建议 380-480ms。移动端底部抽屉逻辑保留。
3. 背景遮罩要轻，不要让页面突然变暗。`rgba(15,23,42,0.24-0.32)` 即可。
4. 弹窗顶部包含国家旗帜/代码、国家名、区域和收入组 badge。保持清晰、紧凑。
5. badge 不要全部蓝色，区域 badge 可以使用对应 WHO 区域色的浅底版本，收入组 badge 使用中性紫灰或绿色系。
6. 弹窗里的 metric cards 不要像重复小按钮，应像 compact factsheet：小标签、数值、单位、左边色条。
7. 弹窗中的当前国家趋势图使用洋红作为主线，历史背景/参考线用灰色。
8. 风险因素条形图要有明确排序，条形颜色可使用风险橙红梯度，但不要和洋红选中态混淆。
9. 资源方案部分要突出“当前/优化/建议调整”的对比，可使用微型差值标签。
10. 关闭按钮要可见、可点、focus 明确，不要只有 hover 才看得见。

动效与顺滑度要求：

1. 动效应该让状态变化更容易理解，而不是展示动画本身。
2. 保留页面加载时的轻微 stagger，但总时长不要超过 500ms。
3. 维度切换时可以有短暂淡出淡入，避免图表突然跳变。
4. 卡片 hover 只做 `translateY(-1px)` 或阴影变化，不要大幅移动。
5. 不要使用无限循环 shimmer、呼吸光、旋转 glow、强粒子、强背景动画。
6. 播放按钮可以在播放状态有非常轻的 pulse，但不能吸引过多注意力。
7. 所有动画都必须尊重 `prefers-reduced-motion`。

字体与排版要求：

1. 中文界面优先使用 `Alibaba PuHuiTi 3.0`、`PingFang SC`、`Noto Sans SC`、系统 sans-serif。
2. 数字使用等宽数字字体或启用 tabular numbers，保证排名、指标、年份稳定。
3. 页面正文基础字号不要低于 13px。重要数值建议 17-22px。
4. 卡片标题 16-18px 即可，不要把内部卡片标题做得像 hero。
5. 字距不要为负，中文大写标签也不要过度 letter-spacing。
6. 所有文本必须避免溢出。长国家名、地区名、省份名使用 ellipsis 或换行策略。
7. 控件里的文字必须完整可读，按钮不能因为中文长度被挤压。

布局和空间要求：

2. 卡片间距保持 14-18px，内部 padding 保持 18-22px，移动端可降到 14px。
3. 不要把卡片嵌套进卡片。需要分组时使用浅底区域或细分割线。
4. 页面最大宽度可以保持约 1560px，但在宽屏下不能显得松散。地图、侧栏、图表应形成稳定网格。
5. 移动端顺序建议：顶部摘要、控件、地图、国家概况、趋势图、比较图、核心洞察。不要让控件栏占满首屏后看不到内容。
6. sticky 控件栏在移动端如果占空间过大，可以取消 sticky 或压缩为更紧凑样式。

数据主题表达要求：

1. 主题不是普通“健康大屏”，而是“全球健康转型、风险因素、卫生资源公平、中国大陆省级差异和健康发展全景”的综合分析。
2. 视觉语言要体现“公平、公共卫生、资源优化、全球对比”，可以用细线、分层、矩阵、经纬网、指标编码表达理性分析感。
3. 中国大陆聚焦维度要体现省际资源配置和公平优化，不要做成行政地图展示而已。
4. 健康全景维度要体现发展水平、环境暴露、服务覆盖和健康结果之间的关联。
5. 风险因素维度可使用更暖的风险色调，但仍需保持专业，不要像警报大屏。

可访问性与可读性要求：

1. 所有交互控件必须有清晰 focus 状态。
2. 文本与背景对比度必须满足普通阅读需求，弱文字也不能低到不可读。
3. 不要仅依靠颜色表达选中状态。选中态需要同时有颜色、边线/形状/图标/左条等辅助线索。
4. 色盲用户也要能分辨当前国家：洋红高亮 + 白色 halo + 线宽变化是必须的。
5. hover tooltip 不应遮挡关键内容过久，内容要简洁。
6. 键盘操作 combobox 时 active row、selected row、focus ring 必须视觉不同。

第一阶段：建立非蓝色高亮系统。将所有“当前国家/省份/当前对象”的蓝色或青色选中状态迁移到洋红 `#D81B60`，包括地图边线、排名选中行、图表当前点、弹窗当前趋势线、国家标题闪动、必要的 focus 辅助状态。确保普通品牌蓝仍可用于导航和控件，但不再表示当前国家。

第四阶段：统一图表语言。统一 Plotly 背景、字体、网格线、hover、当前对象强调、图例位置和颜色语义。

第五阶段：优化 spotlight 弹窗。把它变成国家 factsheet，结构紧凑、层级清楚、趋势和风险解释更容易扫读。

第六阶段：响应式与最终审美校准。检查桌面、平板、手机三个宽度，确保无文字重叠、无控件挤压、无图例遮挡、无卡片高度跳动。

验收标准：

1. 第一眼观感：浅色、专业、精致、可信，主题明确。
2. 地图观感：地图仍是主角，当前国家/省份洋红高亮非常清楚，不与地图色带冲突。
3. 控件体验：维度切换、指标选择、风险因素选择、预算选择、国家/省份搜索都清楚且顺滑。
4. 阅读体验：中文标题、说明、数值、图例、排名均清楚，无重叠和截断异常。
5. 交互反馈：点击地图、点击排名、搜索选择、打开弹窗后，当前对象在所有相关区域同步突出。
6. 动效体验：状态变化有节奏，但没有持续干扰性动画。
7. 响应式体验：桌面和移动端都能完成核心浏览，不因控件或图表挤压导致不可用。
8. 主题表达：评委能理解这是围绕全球健康、疾病谱变迁、风险因素、资源配置、中国省级差异和健康发展全景的综合洞察系统。
